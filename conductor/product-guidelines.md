# User Experience (UX) Principles
- **Efficiency:** Streamlined workflows to minimize manual input.
- **Clarity:** Clear and consistent naming conventions for all artifacts.
- **Traceability:** Maintain links between original inputs and generated backlogs.
- **Actionability:** Ensure all generated tasks are clear and immediately executable.

# Prose & Branding
- **Professional & Concise:** Use direct, senior-level technical language.
- **Technical Accuracy:** Use appropriate project management and Azure DevOps terminology.
- **Bilingual Awareness:** Support and handle inputs/outputs in both English and Spanish as per project usage.

# Documentation Standards
- **Standard Format:** Follow the standard `YYYYMMDD.TˇpicoPrincipal.NN.Etapa.json` naming convention.
- **Modular Documentation:** Use directory-specific `GEMINI.md` files for granular context management.
- **Self-Documenting Code:** Prioritize readable code and comments that explain the *why*, not just the *what*.
