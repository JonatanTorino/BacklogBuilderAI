1) Caso de uso

Objetivo: necesidad + valor funcional
Fórmula Actor + Objetivo + Valor + Gatillos + Flujo + Reglas + Excepciones
Mini‑plantilla
Actor:
Objetivo:
Valor:
Gatillos:
Flujo básico (pasos):
Reglas de negocio:
Excepciones/alternos:
Ejemplo correcto
  • Actor: Admin de licencias
  • Objetivo: Habilitar/bloquear funcionalidades por entidad legal
  • Valor: cumplimiento y control de costos
  • Regla: tres niveles ON/OFF (global, producto, funcionalidad)
Ejemplo incorrecto
  • “Hacer módulo de licencias completo con todo”
  • Problemas: vago, sin actor, sin valor, sin reglas

2) User Story

Azure DevOps: Title • Description • Acceptance Criteria
Título = <Verbo> <objeto> <para/por beneficio>
Descripción
Rol: <actor>
Necesidad: <acción deseada>
Valor: <beneficio>
Acceptance Criteria (Given‑When‑Then)
  • Given estado inicial
  • When acción
  • Then resultado medible
Ejemplo correcto (AxnLM)

Title: Revocar licencia para liberar cupo

Rol: Administrador de licencias
Necesidad: Revocar una licencia activa de un usuario
Valor: Reasignar el cupo sin comprar licencias extra
  • Given usuario A con licencia activa, When presiono “Revocar”, Then la licencia pasa a “Disponible” y se registra auditoría.
  • Given uso activo, When intento revocar, Then se bloquea con motivo.
  • Given “Disponible”, When asigno a B, Then queda “Asignada a B” con trazabilidad.
Ejemplo incorrecto

Title: Gestionar favoritos con filtros y botones

  • Descripción vaga. Impone solución.
  • AC no testables: “que funcione bien”.

3) Task

Azure DevOps: Title • Description • DoD
Principio: cada task cierra ≥ 1 criterio de aceptación
Título = <Verbo imperativo> <objeto> (AC-#)
Descripción: Qué + Cómo mínimo + Validaciones/errores
DoD: Tests pasan + PR revisado + Artefacto actualizado + Deploy a dev
Tamaño: ≤ 1 día. Si no, dividir.
Ejemplo correcto (AxnLM)

Title: Implementar POST /licenses/{id}/revoke (AC-2)

Descripción: Endpoint REST que valida “En uso”, retorna 409 si corresponde, registra evento Revoked en auditoría.

  • DoD: 3 unit tests, OpenAPI actualizado, PR aprobado, deploy a dev, changelog.
Ejemplo incorrecto

Title: Mejorar revocar

  • Descripción ambigua. Sin vínculo a AC.
  • DoD inexistente. Tamaño indefinido.
